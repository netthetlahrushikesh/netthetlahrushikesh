## Flagship Project

### Used Car Price Intelligence Platform

An end-to-end listed-price intelligence system for the Indian used-car market,
built from trusted-source acquisition through production deployment.

[![Used Car Price Intelligence Platform](https://raw.githubusercontent.com/netthetlahrushikesh/Used-Car-Price-Intelligence-Platform/main/docs/assets/stage2-web-revamp-desktop.png)](https://used-car-price-intelligence-platfor.vercel.app)

**What the project includes**

- Source-specific acquisition for Spinny, Mahindra First Choice, and Maruti
  Suzuki True Value.
- Canonical schema, normalization, completeness checks, deduplication, listing
  lifecycle tracking, and source-drift analysis.
- Reproducible EDA, model experiments, segment evaluation, interpretation, and
  repeated-split stability validation.
- A versioned model artifact served through FastAPI with typed request and
  response schemas.
- A responsive pricing workspace deployed to Vercel with automated tests and
  documented limitations.

**Measured release evidence**

| Evidence | Result |
| --- | ---: |
| Combined trusted modeling rows | 9,110 |
| Primary holdout MAPE | **9.88%** |
| Seven-split mean MAPE | **10.33%** |
| Primary holdout MAE | INR 47,389 |
| Automated tests | **192 passed** |

The model estimates listed price, not guaranteed transaction value. Premium
vehicles and rare brand-model groups remain documented improvement areas.

```text
Trusted Sources -> Canonical Data -> Quality Gates -> EDA and Modeling
                -> Versioned Artifact -> FastAPI -> Web App -> Vercel
```

## How I Work

- Start with data quality and source trust before optimizing a model.
- Keep raw observations, unique entities, and modeling rows conceptually
  separate.
- Compare baselines before adding model complexity.
- Evaluate by business-relevant segments, not only one aggregate score.
- Document failed experiments, tradeoffs, and limitations alongside results.
- Move validated models into typed, tested, deployable application boundaries.
